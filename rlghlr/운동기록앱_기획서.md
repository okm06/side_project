# 운동 기록 앱 — 기획서

> 작성 시점 기준 정리본. 첫 버전(MVP) 기획과 이후 확장 계획을 담음.

---

## 1. 한 줄 정의

**"운동 막 시작한 사람이, 쉽게 기록하며 성장을 눈으로 확인하고, 비슷한 사람들과 함께 꾸준히 운동하게 되는 앱."**

- 핵심 단어: **빠르게**, **여러 운동(웨이트·유산소)**, **성장 확인**, **함께**
- 기존 앱과의 차별점: 단순 기록 도구가 아니라 **"초보를 안 그만두게 만드는 앱"**

---

## 2. 타깃

**운동 시작한 지 얼마 안 된 초·중급자**

- 고급자는 이미 스트롱·헤비 등에 정착 → 경쟁 불리
- 초·중급자는 아직 자기 앱이 없고, "쉽고 동기부여 되는" 걸 원함
- 이탈률이 가장 높은 층(3개월 내 포기) → "계속하게 만드는 앱" 수요가 가장 큼

---

## 3. 핵심 매력 (왜 계속 쓰는가)

1. **기록하고 성장을 돌아보는 재미** — 혼자서도 작동하는 가치
3. **비슷한 사람과 함께하는 동기부여** — 이탈을 막는 무기

> ※ 둘 다 최종 정체성에 포함. 단 **①부터 만들고, ③(함께)은 사람이 모인 뒤에 켠다.**
> 매칭은 "닭과 달걀 문제"가 있어 0명에서 시작하면 작동 안 함.
> → 기록 툴로 사람을 모은 뒤, 쌓인 데이터(운동 시간대·종목)로 비슷한 사람을 연결하는 전략.

---

## 4. 기록 방식 (가장 중요한 차별점)

**원칙: 세상에서 제일 빠르게 기록되는 앱**

- **검색형 자동완성**: 인풋에 타이핑 시작 → 운동 DB에서 후보 자동완성 → 탭 한 번으로 선택
  - 순수 자유입력 ❌ (같은 운동이 제각각 저장돼 그래프가 깨짐)
  - 순수 select 박스 ❌ (운동이 너무 많아 스크롤 지옥)
  - → 둘의 장점만: 빠른 입력 + 데이터는 깔끔하게 묶임
- **자주/최근 한 운동**을 큰 버튼으로 상단 노출 → 타이핑 없이 탭만으로 선택
- **입력 타이밍**: 운동 끝나고 한 번에 몰아서 입력 (실시간 입력 아님)
  - → 화면 단순화 가능 (타이머·휴식알림 등 불필요)
  - ⚠️ 단점: 끝나면 귀찮아서 안 적게 됨 → **"지난 기록 그대로 불러와 숫자만 수정"** 기능이 핵심

---

## 5. 운동 DB (앱의 뼈대)

**규모: 첫 버전 50~70개** (욕심내지 않기. 부족하면 사용자 반응 보고 추가)

**채우는 법**: 본인이 직접 리스트업(운동인이라 가장 정확) + AI로 초안 뽑고 검수

**운동 1개당 담을 정보**
- 이름 (예: 벤치프레스)
- 부위 (예: 가슴) — "이번 주 가슴 몇 세트" 분석에 사용
- 운동 타입 (웨이트 / 유산소) — 기록 방식이 갈림
- 기록 방식:
  - **웨이트** → 무게 × 횟수 × 세트
  - **유산소** → 시간 · 거리

> 핵심: **운동 타입마다 기록 화면이 다르다.** 벤치프레스(80kg 10회 3세트) vs 러닝(5km 30분)

---

## 5-1. 데이터 구조 (백엔드의 핵심)

**4개의 덩어리(테이블)로 나눔 — 운동 기록 앱의 표준 구조**

**① 사용자 (users)**
- `사용자ID`
- `지역` (시/구 단위, 예: 인천 남동구) — 매칭(③ 함께)에만 사용
- (나중에 추가: 이메일, 닉네임, 자주 가는 헬스장 등)

**② 운동 종류 (exercises)** — 우리가 만들 50~70개 고정 DB
- `운동ID`, `이름`(벤치프레스), `부위`(가슴), `타입`(웨이트/유산소)
- 앱이 미리 갖고 있는 고정 목록

**③ 운동 세션 (sessions)** — "하루의 운동 한 판"
- `세션ID`, `사용자ID`, `날짜`
- **달력에 점 찍히는 단위가 이것** ("1월 5일에 운동함" = 세션 1개)

**④ 기록 (records)** — 세션 안의 개별 운동 내용
- `기록ID`, `세션ID`, `운동ID`, 타입별 수치
- 웨이트 → `무게`, `횟수`, `세트`
- 유산소 → `시간`, `거리`

**⑤ 루틴 (routines)** — 사용자가 미리 만든 운동 묶음
- `루틴ID`, `사용자ID`, `루틴이름`(예: 가슴 루틴), `운동 목록`(포함된 운동들)
- 마이페이지에서 만들고, 기록할 때 불러오면 운동들이 한 번에 채워짐 → 무게·횟수만 입력
- "빠른 기록"의 핵심 장치 (매번 운동 하나하나 검색 안 해도 됨)

**화면과의 연결**
- 달력 ← ③세션의 `날짜` 모으기
- 운동 상세 ← 특정 세션의 ④기록들
- 그래프 ← ④기록에서 특정 운동만 뽑아 `무게`를 날짜순으로
- 기록 ← 새 세션 생성 후 ④기록 여러 개 추가 (⑤루틴 불러오기로 빠르게)

**세트 저장 방식**
- 처음엔 단순하게: "벤치프레스 80kg 10회 3세트" = 한 줄 (3세트 동일 무게)
- 나중에: 세트별 다른 무게(80→75→70) 지원 시 세트별로 줄 분리

**설정 / 정책**
- **단위: kg 고정** (대한민국 한정)
- **지역: 시/구 단위** (동·GPS는 민감 + 매칭엔 시/구로 충분)
- **회원가입 전략**: 가입 화면은 나중에. 단 **데이터는 처음부터 "사용자ID"에 묶는 구조**로 설계.
  - 처음엔 기기당 임시 ID 자동 발급 → 나중에 회원가입 붙이면 임시 ID를 계정에 연결만
  - 이유: 나중에 "쌓인 기록을 계정에 옮기는" 골치 방지 + 매칭(③)은 계정이 있어야 작동
  - 지역도 같은 논리: 구조엔 미리 칸 만들고, 입력 화면은 매칭 붙일 때 제작

## 6. 첫 버전 화면 구성 (4개)

1. **기록 화면** — 운동 끝나고 빠르게 입력 (자동완성 + 지난 기록 불러오기)
2. **달력 화면** — 어떤 날 운동했는지 한눈에, 운동한 날 표시 (꾸준함 동기부여)
3. **그래프/통계 화면** — 운동별 추이로 성장 확인
   - 웨이트: 무게 추이 (또는 볼륨 = 무게×횟수×세트)
   - 유산소: 거리·페이스 추이
4. **운동 상세 화면** — 달력에서 특정 날 누르면 그날 뭐 했는지 자세히

> 초보 타깃이라 화면은 이 정도가 적정. 더 늘리면 복잡해서 안 씀.

---

## 6-1. 메뉴 구조 & 메인 화면

**하단 탭바 (4개)**

`[ 홈 ] [ 기록 ] [ 그래프 ] [ 마이 ]`

- **홈** — 첫 화면(메인). 오늘 요약 + 이번 달 달력
- **기록** — 운동 기록하기 (웨이트/유산소 모두 여기서. 운동 고르면 입력 칸이 자동으로 갈림)
- **그래프** — 운동별 추이로 성장 확인
- **마이** — 내 정보·설정 (나중에 지역·매칭·구독 등이 여기 붙음)

**중요 원칙**
- **웨이트/유산소를 메뉴로 나누지 않는다.** 둘은 "운동 종류"지 "화면"이 아님.
  → 기록할 때 운동을 고르면 자동으로 입력 칸이 갈림(벤치프레스=무게·횟수 / 러닝=시간·거리).
  → 메뉴로 나누면 한 날 두 번 기록해야 해서 "한 곳에서 빠르게"가 깨짐.
- **매칭(함께)은 첫 버전 메뉴에 없음.** 나중에 5번째 탭으로 추가하거나 홈에 편입. 지금은 자리만.

**메인 = 홈 화면 구성** (위 → 아래)
1. 인사/날짜 + 테마(다크/라이트) 토글
2. **오늘 요약 + 기록하기 버튼** (홈의 주인공) — 운동 했으면 요약, 안 했으면 "오늘 운동 기록하기" 큰 버튼
3. 이번 달 달력 (운동한 날 표시 + "이번 달 N회")
4. 짧은 통계 (이번 주 N회 등) — 그래프 맛보기, 선택
- 추천 루틴은 홈에 ❌ → 루틴(마이) 화면에. 홈은 "나의 기록"에 집중.
- 이유: 초보의 1순위 이탈 원인이 "동기 저하" → 켜자마자 쌓인 기록이 보이면 강력한 동기부여.

> 참고: 화면 4개 중 **운동 상세**는 달력에서 특정 날을 눌러 진입하는 화면이라 메뉴에 없음.

- **다크 ↔ 라이트 테마 토글** 둘 다 지원

## 8. 수익 모델 (나중에 켠다)

> ⚠️ 첫 버전은 돈 버는 기능 ❌. "사람이 계속 쓰고 싶은 무료 앱"에 집중.
> 사용자 모인 뒤에 수익화가 작동함.

**기본 골격: 프리미엄(Freemium) 구독**
- 무료: 기본 기록, 최근 기록 보기 등 (충분히 가치 있게)
- 유료(월 3,000~9,000원대): 전체 기록 보관, 상세 분석·통계, 무제한 루틴, 정밀 매칭 필터 등

**추가 수익원**
- **제휴 마케팅**: 운동 장비·꿀팁 추천 → 쿠팡 파트너스 등 수수료 (콘텐츠가 수익으로)
- **B2B(헬스장 제휴)**: 규모 커진 뒤 보너스 트랙

**단계**
1. 무료로 끝내주는 기록 → 사람 모으기 (0원)
2. 사람 쌓이면 → 제휴 링크 + 구독
3. 규모 커지면 → 매칭 고급기능, B2B

## 9. 나중에 붙일 기능 (순서대로) — "버린 게 아니라 미룬 것"

1. **함께 (비슷한 사람 매칭)** — 운동 시간대·종목 데이터로 연결. ③ 핵심 매력의 실현
2. **루틴 공유 피드** — 사용자가 자기 루틴을 공개로 올리고, 인기순(많이 담아간 순)·좋아요·검색
   - 첫 버전엔 "본인이 미리 만든 공식 추천 루틴"만 둘러보기/담기 가능 (빈 화면 방지)
   - 루틴(⑤)에 `공개여부`, `원본 루틴ID` 칸만 미리 만들어두기
3. **인바디(체성분) — 수기 입력만** → 체성분 변화 그래프 (운동 그래프와 나란히)
   - 사진 첨부 / 사진 OCR 자동기록은 **제외** (결과지 양식 제각각, 오류 시 신뢰 손상, AI 비용)
4. **수영** — 기록 방식이 또 달라(레인·영법) 화면 추가 필요. 사용자 비율도 낮아 후순위
5. **자세 교정** — 영상분석/전문가 검수 필요, 잘못되면 부상 위험 → 첫 작품엔 부담
6. **꿀팁·운동장비 정보** — 제휴 마케팅과 연결되는 콘텐츠 영역

## 10. 기술 스택 (참고)

- 프론트: HTML/CSS/JS (퍼블리싱 가능) + AI 보조로 React/Next 등
- 백엔드: **Supabase/Firebase (BaaS)** 권장 — DB·인증·저장소를 직접 서버 안 짜고 해결
- 결제(나중): 토스페이먼츠 / 스트라이프
- ⚠️ AI가 코드 90%는 대신해주지만, DB 설계·인증/보안·배포·에러 디버깅의 "왜"는 본인이 이해해야 막혔을 때 빠져나옴

## 핵심 원칙 (절대 잊지 말 것)

> **좋은 아이디어를 다 넣은 앱이 아니라, 한 가지(빠른 운동 기록)를 끝내주게 하는 앱이 사람을 모은다.**
> 혼자 만들 때 살아남는 유일한 방법 = 순서를 지키는 것.

## 아직 안 정한 것 (시안 만들면서 같이 정할 항목)

- **루틴/프로그램 개념** — "내 가슴 루틴" 묶음을 만들어 불러올지 vs 매번 검색할지.
  "빠른 기록"과 직결되므로 시안 단계에서 결정 필요 (★ 우선순위 높음)
- **온보딩 / 첫 화면** — 앱 처음 켰을 때 뭘 보여줄지
- **빈 화면(Empty State)** — 기록 0개일 때 그래프·달력에 뭘 보여줄지 (초보 앱에 중요)
