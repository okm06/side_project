# 1단계: 탭바 + 화면 전환 (리액트 뼈대)

## 이번 단계에서 만든 것

하단 탭 4개(홈·그래프·기록·마이)를 누르면 화면이 바뀌는 뼈대.

- 다크/라이트 테마 토글.
  페이지 속은 비워뒀고, 다음 단계에서 채울 예정.

## 파일을 어디에 넣나

Vite 프로젝트의 `src/` 폴더에 이렇게 배치:

```
src/
├─ App.jsx              ← 받은 파일로 교체
├─ App.css             ← 받은 파일로 교체
├─ components/
│  └─ TabBar.jsx        ← 새로 추가 (폴더 없으면 만들기)
└─ pages/
   ├─ HomePage.jsx      ← 새로 추가
   ├─ RecordPage.jsx    ← 새로 추가
   ├─ GraphPage.jsx     ← 새로 추가
   └─ MyPage.jsx        ← 새로 추가
```

`main.jsx`는 Vite 기본 그대로 두면 됨 (App을 불러오는 부분).
혹시 `index.css`에 기본 스타일이 있으면 비워도 됨 (App.css가 다 처리).

## 실행

```
npm run dev
```

브라우저에서 열고 → 하단 탭 눌러서 화면 바뀌는지, 🌙 눌러서 테마 바뀌는지 확인.

## 이번에 배우는 리액트 개념

1. **useState** — "바뀌는 값"을 저장 (현재 탭, 테마)
2. **props** — 부모(App)가 자식(TabBar, 페이지)에게 값·함수를 넘김
3. **조건부 렌더링** — activeTab 값에 따라 알맞은 페이지만 그림
4. **map으로 목록 그리기** — 탭 배열을 반복해서 탭 버튼 생성
5. **컴포넌트 분리** — 화면을 파일(조각)로 나눠 관리

## 흐름 한눈에

```
탭 누름
 → TabBar가 onTabChange(id) 호출
 → App의 setActiveTab(id) 실행
 → activeTab 값 바뀜
 → 리액트가 자동으로 해당 페이지 다시 그림
```

직접 화면을 숨기고 보이는 게 아니라, "값을 바꾸면 화면이 따라온다"가 핵심.
